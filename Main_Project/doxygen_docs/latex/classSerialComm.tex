\doxysubsection{Detailed Description}
This is the class used for all serial-\/related operations in the application. 

This is the Class that is used solely and exclusively by the application for any and all operations involving serial read, serial write, or any serial configuration. It is the single point of access that all other entities should use for any serial operations.

This class specifically, handles configuring, opening, and closing serial ports, and exposes public methods for sending commands to the microcontroller to modify its runtime behavior.

Performs a handshake with a device over a serial port. 

This function attempts to establish communication with a device connected to the specified serial port by sending a "{}ping\textbackslash{}n"{} message and waiting for a "{}pong\textbackslash{}n"{} response.

The function will:\+
\begin{DoxyItemize}
\item Open the specified serial port
\item Send a "{}ping\textbackslash{}n"{} message
\item Wait (with timeout) for a response
\item Read incoming data until a newline character is encountered
\item Compare the received message against the expected "{}pong"{}
\end{DoxyItemize}

This function reads a single dataframe, parses it, and assigns readings to \doxylink{classSensor}{Sensor} objects. 

When this function is called, it reads from the serial input buffer exactly one dataframe, which is demarcated by a newline character. The dataframe coming from the microcontroller contains comma separated values, which are indexed according the order of \doxylink{classSensor}{Sensor} objects stored in the vector in \doxylink{classProjectPanel}{Project\+Panel}.

The dataframe is parsed, and each \doxylink{classSensor}{Sensor} is assigned its corresponding reading.

This method is designed to run at a rapid rate on a background polling thread. It simply reads a dataframe, parses it, and assigns values to each \doxylink{classSensor}{Sensor} that has been configured in a project. It does not care what is done with those readings -\/ it simply updates them.
