\doxysubsection{Detailed Description}
This is the GUI object that wraps around all data and configurations of a project. 

When a user sets up or loads a project, this is the main class around which all activities revolve. In terms of ownership, nearly every other class can be traced back to this. Most in-\/project events are handled within this class, which is something that we are looking at reevaluating. In the future, we may delegate more event handlers to more specialized classes, but it is not expected that the class ownership hierarchy will change.

void Project\+Panel::\+toggle\+Start\+Stop (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})}



Toggle function used to control sensor data recording sessions. 

This method is called when the user clicks the Start/\+\+Stop toggle button located in \doxylink{classToolbar}{Toolbar}. When called, it checks to see if a run is already in session, and if so, stops it. If there is no active running recording session, it will create a new one.

\begin{DoxyNote}{Note}
If a new recording session is created, it will do so without destroying any previous runs. Unless the reset\+Session\+Data method is invoked, all runs of data will be retained by default, so that multiple runs can be recorded, compared, and/\+or exported by the user. 
\end{DoxyNote}
