\doxysubsection{Detailed Description}
Manages all \doxylink{classSensor}{Sensor} related modifications in a single \doxylink{classProjectPanel}{Project\+Panel}. 

There is exactly one \doxylink{classSensorManager}{Sensor\+Manager} per project. \doxylink{classSensor}{Sensor} Objects are stored in a vector, owned by \doxylink{classProjectPanel}{Project\+Panel}, and whenever a sensor itself is modified -\/ added, removed, calibrated, etc -\/ the \doxylink{classSensorManager}{Sensor\+Manager} is used to handle this.

\doxylink{classSensorManager}{Sensor\+Manager} performs intermediate logical operations, like parameter validation. It will not allow the vector of \doxylink{classSensor}{Sensor} objects to be modified under erroneous parameters.

\begin{DoxyNote}{Note}
While modifications to the vector of \doxylink{classSensor}{Sensor} Objects is strictly controlled by \doxylink{classSensorManager}{Sensor\+Manager}, access to the individual \doxylink{classSensor}{Sensor} objects themselves is allowed from anywhere in the application. Typically, this is accomplished by passing raw pointers to the vector -\/ which is owned by \doxylink{classProjectPanel}{Project\+Panel} via unique\+\_\+pointer -\/ to anywhere that needs to actually use a \doxylink{classSensor}{Sensor}. 
\end{DoxyNote}
