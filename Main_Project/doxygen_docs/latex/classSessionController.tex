\doxysubsection{Detailed Description}
Manages the polling thread m\+\_\+thread, and handles clearing data from each \doxylink{classSensor}{Sensor} \doxylink{classDataSession}{Data\+Session}. 

\doxylink{classSessionController}{Session\+Controller} is responsible for start/\+stop/\+reset lifecycle handling. It doesn\textquotesingle{}t itself poll the serial port. It creates a background thread that makes timed calls to \doxylink{classSerialComm}{Serial\+Comm} for that. Instead, it manages the "{}running"{} state flag, reset logic (cleaning sessions), and exposes a polling rate adjustment API for the thread, which will run at the specified rate.
