The purpose of this text file is to describe the behavior of the class SerialComm.

The purpose of the class is to provide a modular interface for interacting with serial port devices in
a larger program. 

GENERAL DESCRIPTION:

	- First, the class is instantiated. No parameters are set, but there is a default configuration:
		-BAUD:		9600
		-DataBits:	8
		-Parity:	none
		-Stop:		1
		-FlowControl:	None	

	- once the class is instantiated, it will auto-generate a list of available ports. this information 
		will be stored in a string array.

	- from inside an opened project in the main application, the user will be able to select one of
		these ports from a drop-down list. In this same menu, there will be an "advanced" tab used
		to manually enter parameters. The default parameters will be auto-loaded, but may be changed
		according to the user's preferences. These values will also be in the form of drop-down 
		menus to avoid allowing users to enter parameters that result in undefined behavior.

	- once the port is selected, the SerialComm class will establish a connection with the specified port
		using the default parameters, or the user-specified parameters if changed. 

	- once an established connection exists, the class SerialComm will perform a handshake with the 
		connected serial device. The number of sensors being used must be agreed upon by both the
		class and the connected device. 

	- once the handshake is successful, the connected device will wait until it is sent a start signal.

	- upon receiving the start signal, the device will begin streaming packets of data through the port. 
		after each packet, it will check for a stop signal. If it recieves this signal, it will
		continuously check for another start signal, not sending further data until it is recieved.

		A single data packet will be structured as:

		|start|Sensor1|Value|Sensor2|Value|Sensor...|Value|End|
